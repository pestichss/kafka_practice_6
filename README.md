# Практическая работа №6: Настройка защищённого соединения и управление доступом в Apache Kafka

## 🎯 Цель работы
Развернуть защищённый отказоустойчивый кластер Apache Kafka из трёх брокеров и Zookeeper с использованием Docker Compose. Настроить сквозное шифрование данных по протоколу SSL/TLS (mTLS) и разграничить права доступа для клиентов на уровне логики приложения (ACL).

## 🛠️ Инструкция по запуску и проверке

### 1. Подготовка окружения
Убедитесь, что у вас запущен **Docker Desktop** и установлена библиотека для работы с Kafka в Python:
```bash
pip install confluent-kafka
```

### 2. Запуск кластера Kafka
Запустите кластер из трёх брокеров в фоновом режиме:
```bash
docker compose up -d
```
*Пожалуйста, подождите 15-20 секунд после запуска, чтобы брокеры успели обменяться метаданными по защищённым каналам.*

Проверить статус контейнеров можно командой:
```bash
docker compose ps
```
*Все 4 сервиса должны находиться в статусе `Up`.*

### 3. Автоматическое тестирование через Python
Запустите встроенный проверочный скрипт, который выполнит отправку и чтение зашифрованных сообщений:
```bash
python verify_security.py
```

### 🎯 Ожидаемый результат проверки (Лог работы)
Скрипт проверяет безопасность кластера в два этапа:
1. **Топик `topic-1` (Доступ разрешён всем)** — Продюсер успешно шифрует и отправляет сообщение, Консьюмер успешно читает его.
2. **Топик `topic-2` (Чтение ЗАПРЕЩЕНО)** — Продюсер успешно записывает данные, а при попытке чтения Консьюмером срабатывает блокировка доступа.
